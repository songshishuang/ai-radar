## 🔧 GitHub 高星开源盘点（产品 × 技术双视角）

> ⭐ 为 GitHub API 实时星标 · 均近期活跃（pushed ≥6-15）

**① [ollama/ollama](https://github.com/ollama/ollama)** ⭐164k · Go · `本地运行`
- 🎯 **产品**：一行命令在本地跑开源大模型（Kimi-K2.6 / GLM-5.1 / DeepSeek / MiniMax 等），零云依赖、零 API key。
- 🔧 **技术**：轻量 Go 运行时 + 模型量化 + 统一拉取接口，Mac/Linux/Win 全平台。
- 💡 **为何现在看**：开源迁移主线下的「本地运行事实标准入口」——你做开源模型实测，第一站就是它。

**② [browser-use/browser-use](https://github.com/browser-use/browser-use)** ⭐100k · Python · `Agent 操作`
- 🎯 **产品**：让 AI agent 像人一样操作网页（点击/填表/抓取），**无 API 也能自动化**。
- 🔧 **技术**：浏览器自动化 + DOM/视觉理解 + LLM 决策闭环。
- 💡 **为何现在看**：正是今天 Gemini computer use 下放的**开源对应物**——computer use 的能力，开源早有实现。

**③ [firecrawl/firecrawl](https://github.com/firecrawl/firecrawl)** ⭐138k · TypeScript · `数据基建`
- 🎯 **产品**：把任意网站抓成 LLM 友好的干净数据，给 RAG / agent 喂料的「数据上游」。
- 🔧 **技术**：抓取 + 渲染 + 解析 + 清洗全 API 化，处理 JS 动态站。
- 💡 **为何现在看**：RAG/agent 质量的天花板在数据，firecrawl 是这条链路最成熟的开源基建。

**④ [langchain-ai/langchain](https://github.com/langchain-ai/langchain)** ⭐140k · Python · `Agent 框架`
- 🎯 **产品**：构建 LLM 应用 / agent 的工程平台（编排 + 工具调用 + 记忆）。
- 🔧 **技术**：链式编排 + **厂商无关抽象层**（一套代码接所有模型）。
- 💡 **为何现在看**：「厂商无关」正是对冲供应商风险的工程基本功，呼应开源迁移要的「换模型=改配置」。

**⑤ [dyad-sh/dyad](https://github.com/dyad-sh/dyad)** ⭐20k · TypeScript · `AI 应用生成`
- 🎯 **产品**：**本地、开源**的 AI app builder（v0 / Lovable 的本地替代），数据不出本机。
- 🔧 **技术**：本地运行的代码生成 + 预览，隐私可控。
- 💡 **为何现在看**：AI 编程工具的「本地+开源」路线，对隐私/合规敏感团队是 SaaS 版的替代。

**⑥ [screenpipe/screenpipe](https://github.com/mediar-ai/screenpipe)** ⭐19k · Rust · `个人 AI`
- 🎯 **产品**：本地记录你「看过/说过/听过」的一切，做个人 AI 记忆与上下文。
- 🔧 **技术**：Rust 本地录制 + **本地 ML 推理**（数据不出本机）。
- 💡 **为何现在看**：个人 AI + 隐私优先的代表，和 Google Cloud 可验证私密 AI 是同一条「可信 AI」主线的两端（企业级 vs 个人级）。

**给 PM 的一句话归纳**：这六个项目其实指向同一个趋势——**AI 能力正在「本地化 + 开源化 + 可控化」**（ollama 本地跑、dyad 本地建、screenpipe 本地记），叠加今天的「欧洲算力主权」「开源迁移共识」，**「可自部署 / 不依赖单一云厂商」从理念变成了成熟工具链**。做选型与架构时，这条路线的现成弹药比想象的足。
