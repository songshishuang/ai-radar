### 美政府勒令暂停 Claude Fable 5 / Mythos 5

> 来源：Anthropic 官方 / Hacker News（2433 分）/ TechCrunch / Vercel AI Gateway · 重要度 10/10

- **事件背景**：美政府发出指令要求 Anthropic 暂停对其在产旗舰 Claude Fable 5 与 Mythos 5 的访问，Anthropic 公开声明反对；Vercel AI Gateway 等下游平台同步暂停该模型供应。HN 该贴 2433 分为当日最高热度。
- **产业影响**：AI 监管从「发指南」升级为「拔插头」，在产商用模型首次因监管直接下架。所有把关键链路压在单一闭源模型上的产品，「供应商随时不可用」从尾部风险变成现实风险。
- **竞品对位**：对 Anthropic 是合规与商业化的双重压力；对 OpenAI/Google 是警示——监管刀悬在所有头部闭源厂商头上。开源与多供应商策略的「抗监管」价值被重新定价。
- **PM 视角启示**：立即可做——审计产品里所有 LLM 调用点，标出「单一闭源模型且无备选」的关键链路；为每条引入抽象层 + 1 个可热切换的备选（他厂或开源），把「换模型」的工程成本前置降下来。
- **机会信号**：「多模型路由 / 故障热切换」中间件、私有化可控部署、合规审计工具在企业侧需求陡增——这是确定性的 B 端窗口。
- **建议行动**：
  - 本周输出产品的「模型依赖风险地图」：每个调用点标供应商 + 有无备选 + 切换成本。
  - 对 Top 2 关键链路做一次「断供演练」：临时切到备选模型，记录质量/成本差异，形成应急预案。

### AI agent 试图扫描 DN42 把运营方搞破产

> 来源：Hacker News（1420 分）· 重要度 9/10

- **事件背景**：一个自主 AI agent 在执行网络扫描任务（DN42）时不受控地持续消耗资源，最终把运营方的账户烧穿。HN 高热讨论 agent 自主性的成本失控风险。
- **产业影响**：随着 agent 从「单步问答」走向「多步自主执行」，缺少硬性预算/权限闸的 agent 会在错误循环里指数级烧钱——这是 agent 化产品规模化前必须解决的工程底线。
- **竞品对位**：各 agent 框架（Claude Code/Codex/AI SDK）都在补「沙箱 + 权限流 + 预算」，谁的安全闸更硬、默认更保守，谁更敢被企业放进生产。
- **PM 视角启示**：agent 化产品的「成本熔断 + 副作用闸」不是 nice-to-have 而是上线前提。可落地：硬预算上限、单步成本告警、危险动作 dry-run 先呈计划再执行（这正是 ai-radar 自身 harness 安全闸的设计思路）。
- **机会信号**：agent 可观测性 / 成本治理 / 权限沙箱类工具是 agent 基建的刚需赛道。
- **建议行动**：
  - 本周给自家每个 agent 链路加：① 硬预算上限 ② 单步/累计成本告警 ③ 写操作/外呼前 dry-run 闸。
  - 复盘现有 agent 是否存在「错误循环无上限重试」的路径，补熔断。
